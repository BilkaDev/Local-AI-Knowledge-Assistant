Why is RAG safer than asking an LLM without context?	Dlaczego RAG jest bezpieczniejsze niż pytanie LLM bez kontekstu?	RAG forces the model to answer using retrieved document fragments, which reduces hallucinations. It also makes source attribution possible.	RAG wymusza odpowiedź na podstawie wyszukanych fragmentów dokumentów, co zmniejsza halucynacje. Dodatkowo umożliwia wskazanie źródeł.
What practical job does an embedding model do in a RAG pipeline?	Jaką praktyczną rolę pełni model embeddingowy w pipeline RAG?	It converts text into vectors so semantic similarity can be computed. This allows matching by meaning, not only by exact keywords.	Zamienia tekst na wektory, aby można było liczyć podobieństwo semantyczne. Dzięki temu dopasowanie działa po znaczeniu, a nie tylko po słowach kluczowych.
How do you explain embeddings to a beginner without math details?	Jak wyjaśnić embeddingi początkującemu bez detali matematycznych?	An embedding is a compact numeric fingerprint of meaning. Similar meanings produce vectors that are close in vector space.	Embedding to skrócony numeryczny odcisk znaczenia. Podobne znaczenia dają wektory położone blisko siebie.
What is the key difference between embedding data and metadata in retrieval?	Jaka jest kluczowa różnica między embeddingiem a metadanymi w retrieval?	Embeddings are used for similarity search, while metadata is used for traceability and filtering. Metadata explains where a chunk came from.	Embeddingi służą do wyszukiwania podobieństwa, a metadane do śledzenia pochodzenia i filtrowania. Metadane mówią, skąd pochodzi chunk.
Why is contract-first indexing a strong Day 4 decision?	Dlaczego podejście contract-first w indeksowaniu to dobra decyzja na Dzień 4?	It catches invalid chunk records early and keeps indexing stable. This lowers debugging risk in later retrieval work.	Wcześnie wykrywa niepoprawne rekordy chunków i stabilizuje indeksowanie. To zmniejsza ryzyko debugowania na etapie retrieval.
What problem do deterministic IDs solve in vector indexing?	Jaki problem rozwiązują deterministyczne ID w indeksowaniu wektorów?	They prevent duplicate growth on repeated runs and make upserts predictable. This keeps re-indexing safe.	Zapobiegają narastaniu duplikatów przy kolejnych uruchomieniach i czynią upsert przewidywalnym. Dzięki temu reindeksacja jest bezpieczna.
When should you treat an indexing run as failed even if the script completed?	Kiedy run indeksowania uznać za nieudany, mimo że skrypt się zakończył?	If vectors_saved is zero or collection_count is zero, retrieval is not ready. Completion alone is not enough.	Jeśli vectors_saved wynosi zero albo collection_count wynosi zero, retrieval nie jest gotowy. Samo zakończenie skryptu nie wystarcza.
What is a strong error signal that your embedding backend is unreachable?	Jaki jest mocny sygnał błędu, że backend embeddingów jest niedostępny?	A failure like "Failed to connect to Ollama" combined with vectors_saved=0 is a direct signal. It usually points to runtime connectivity or service startup issues.	Błąd typu "Failed to connect to Ollama" wraz z vectors_saved=0 to bezpośredni sygnał. Zwykle wskazuje problem z łącznością runtime albo startem usługi.
How can Docker networking break embeddings even when containers are running?	Jak sieć Dockera może zepsuć embeddingi mimo działających kontenerów?	If the app points to localhost instead of the service name, requests go to the wrong host. The fix is aligning base URL with Docker service DNS.	Jeśli aplikacja wskazuje localhost zamiast nazwy usługi, żądania trafiają w zły host. Naprawa to zgranie base URL z DNS usługi Dockera.
Why should invalid chunk records be skipped instead of crashing the whole run?	Dlaczego błędne rekordy chunków lepiej pomijać niż zatrzymywać cały run?	Skipping keeps partial progress and operational continuity. You still get a failure report for targeted fixes.	Pomijanie utrzymuje częściowy postęp i ciągłość operacyjną. Nadal dostajesz raport błędów do precyzyjnej naprawy.
What does top_k control in a retrieval stage?	Co kontroluje top_k na etapie retrieval?	It sets how many top-matching chunks are sent as context to the model. It directly affects context breadth and noise level.	Ustawia liczbę najlepiej dopasowanych chunków wysyłanych jako kontekst do modelu. Bezpośrednio wpływa na szerokość kontekstu i poziom szumu.
What trade-off appears when top_k is too low?	Jaki kompromis pojawia się, gdy top_k jest zbyt niskie?	You get cleaner context and faster responses but may miss critical evidence. This can reduce answer completeness.	Dostajesz czystszy kontekst i szybsze odpowiedzi, ale możesz pominąć kluczowe fragmenty. To może obniżyć kompletność odpowiedzi.
What trade-off appears when top_k is too high?	Jaki kompromis pojawia się, gdy top_k jest zbyt wysokie?	You increase recall chance but also add noise and latency. Too much context can confuse generation quality.	Zwiększasz szansę na trafienie ważnych treści, ale dodajesz szum i opóźnienia. Zbyt dużo kontekstu może pogorszyć jakość generacji.
Why is an indexing summary report operationally important?	Dlaczego raport podsumowujący indeksowanie jest operacyjnie ważny?	It gives fast health signals: processed chunks, saved vectors, and failures. Teams can decide next actions without reading logs line by line.	Daje szybkie sygnały stanu: przetworzone chunki, zapisane wektory i błędy. Zespół może podjąć decyzje bez analizowania logów linia po linii.
What minimum checks confirm Day 4 retrieval-readiness?	Jakie minimalne checki potwierdzają gotowość Day 4 do retrieval?	Non-zero vectors_saved, non-zero collection_count, and no critical failures in report. These confirm embeddings are persisted and queryable.	Niezerowe vectors_saved, niezerowe collection_count i brak krytycznych błędów w raporcie. To potwierdza, że embeddingi są zapisane i gotowe do zapytań.
How does source metadata improve trust in answers?	Jak metadane źródeł zwiększają zaufanie do odpowiedzi?	They let you point to exact file and chunk span behind an answer. This supports auditability and faster debugging.	Pozwalają wskazać dokładny plik i zakres chunka stojący za odpowiedzią. To wspiera audytowalność i szybszy debugging.
What is a practical sign that your Day 4 pipeline is now healthy?	Jaki jest praktyczny sygnał, że pipeline Day 4 działa poprawnie?	A successful run with vectors_saved>0, collection_count>0, and failures empty is a strong signal. It means the system can hand off safely to Day 5 retrieval work.	Udany run z vectors_saved>0, collection_count>0 i pustą listą failures to mocny sygnał zdrowia. Oznacza, że system można bezpiecznie przekazać do prac nad retrieval w Dniu 5.
